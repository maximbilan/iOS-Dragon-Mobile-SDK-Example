SKRecognizerDelegate protocol reference

SKRecognizer.h

Overview

The SKRecognizerDelegate protocol defines the messages sent to a delegate of the SKRecognizer class. These delegate methods indicate the flow of the recognition process. The receiver will be notified when the recording has ended and when the recognition process is finished.

Tasks

Managing Recognition State

  • - recognizerDidBeginRecording: Sent when the recognizer starts recording audio.
  • - recognizerDidFinishRecording: Sent when the recognizer stops recording audio.
  • - recognizer:didFinishWithResults: Sent when the recognition process completes successfully.
  • - recognizer:didFinishWithError:suggestion: Sent when the recognition process completes with an error.

recognizer:didFinishWithResults:

Sent when the recognition process completes successfully.

- (void)recognizer:(SKRecognizer *) recognizer didFinishWithResults:(SKRecognition *) results
Parameters
recognizer

The recognizer sending the message.

results

The SKRecognition object containing the recognition results.

Discussion

This method is only called when the recognition process completes successfully. The results object contains an array of possible results, with the best result at index 0 or an empty array if no error occurred but no speech was detected.

recognizer:didFinishWithError:suggestion:

Sent when the recognition process completes with an error.

- (void)recognizer:(SKRecognizer *) recognizer didFinishWithError:(NSError *) error suggestion:(NSString *) suggestion
Parameters
recognizer

The recognizer sending the message.

error

The recognition error. Possible numeric values for the SKSpeechErrorDomain are listed in SpeechKitError.h and a text description is available via the localizedDescription method.

suggestion

This is a suggestion to the user about how he or she can improve recognition performance and is based on the audio received. Examples include moving to a less noisy location if the environment is extremely noisy, or waiting a bit longer to start speaking if the beeginning of the recording seems truncated. Results are often still present and may still be of useful quality.

Discussion

This method is called when the recognition process results in an error due to any number of circumstances. The audio system may fail to initialize, the server connection may be disrupted or a parameter specified during initialization, such as language or authentication information was invalid.
